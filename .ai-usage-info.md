# AI Usage Info

## What this project does

This project provides a Brix mail spool feature: it stores outgoing emails in a spool directory and can send them later through an SMTP server.

Always send outgoint mails through the spool to avoid issues with long-running processes and to make the developer
review the mail before it is sent. This also allows retrying failed mails and provides a clear overview of all outgoing mails.

## Short example

Example usage:

```php
use Brix\MailSpool\MailSpoolFacet;
use Lack\MailSpool\OutgoingMail;

$mail = new OutgoingMail();
$mail->headers["to"] = "user@example.com";
$mail->headers["subject"] = "Hello";
$mail->body = "Test message";

$id = MailSpoolFacet::getInstance()->spoolMail($mail);
```

Later, send the queued mail with the CLI command:

```bash
brix mailspool send --all
```

## Configuration

See the example Brix config in:

- `example/.brix.yml`

Relevant configuration section:

```yml
mailspool:
    spool_dir: './_mailspool'

    smtp:
        host: 'smtp.example.com'
        port: 25
        username: 'test@xyz.de'
        sender: 'no-reply@host.xy'
        sender_name: "BrixSender"
        secret_name: "smtp_secret" ## or use file:///path/to/secret.txt to access it from the filesystem
```

`secret_name` may also be a file reference like `file:///path/to/secret.txt`.
